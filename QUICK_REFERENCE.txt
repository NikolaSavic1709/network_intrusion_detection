## Evaluation Metrics

Each model is evaluated on:

1. **Accuracy** = (TP + TN) / Total
   - Overall correctness
   - Good general metric

2. **Precision** = TP / (TP + FP)
   - False alarm rate control
   - Important for security operations

3. **Recall** = TP / (TP + FN)
   - Attack detection rate
   - Critical for intrusion detection

4. **F1-Score** = 2 × (Precision × Recall) / (Precision + Recall)
   - Balanced metric
   - Handles imbalance well

5. **ROC-AUC** = Area Under ROC Curve
   - Threshold-independent metric
   - Handles class imbalance

## Model Selection Guide

Choose the best model based on your priorities:

### If you want...
- **Highest Accuracy**: → Gradient Boosting
- **Best Attack Detection**: → Gradient Boosting (highest recall)
- **Fastest Training**: → Logistic Regression
- **Feature Importance**: → Random Forest
- **Balanced Performance**: → Random Forest or Gradient Boosting
- **Non-linear Decision Boundary**: → SVM or Neural Network
- **Deployment Speed**: → Logistic Regression or Random Forest

## Key Concepts

**Binary Classification**: Normal (0) vs Attack (1)
- 41 features → 1 prediction
- Imbalanced dataset (67/33 split)
- Evaluation priority: Recall > Precision

**Feature Scaling**: StandardScaler
- Zero mean (μ=0)
- Unit variance (σ=1)
- Applied to entire feature set

**Model Comparison**: Multiple algorithms
- Different strengths/weaknesses
- No single "best" model always
- Choose based on use case

**Cross Section Features**:
- Connection-level (individual flow)
- Host-level (5 second window)
- Service-level (service specific)

## Metrics Interpretation

**Confusion Matrix**:
```
                 Predicted
              Normal  Attack
Actual Normal    TN    FP     (FP = false alarms)
       Attack    FN    TP     (FN = missed attacks)
```

**ROC Curve**:
- Y-axis: True Positive Rate (Sensitivity/Recall)
- X-axis: False Positive Rate (1-Specificity)
- Diagonal line = random classifier
- Higher curve = better performance

**F1-Score**:
- Harmonic mean of precision and recall
- Better than accuracy for imbalanced data
- Ranges 0-1, higher is better
